# 安全境界

NeonSec Academyは、認可された防御セキュリティ学習のための学習ツールです。その内容は、実際のシステムに対する活動を可能にすることなく、認識、推論、検出、優先順位付け、修正、報告を教える必要があります。

## コアポリシー

使用するのは以下のみ：

- このリポジトリに含まれるシード質問。
- Safe Labsに付属する合成証拠。
- 秘密や個人データを含まないローカルノート、レポート、ブラウザデータ。
- このポリシーに従ったユーザー作成のコンテンツ。
- 架空のホスト、架空のアカウント、文書IP範囲、おもちゃのスニペット。

使用してはいけないもの：

- 実際の公開IP、実際のドメイン、プロダクションホスト、サードパーティサービス、または外部ターゲット。
- 実際の認証情報、トークン、セッションクッキー、秘密鍵、APIキー、パスワードダンプ、または顧客データ。
- ライブマルウェア、エクスプロイトペイロード、フィッシング配信、認証情報収集、トラフィック生成、またはサービス拒否活動。
- システムを侵害する方法、検出を回避する方法、持続性を維持する方法、データを外部に持ち出す方法、またはアクセス制御をバイパスする方法を説明する指示。

リスクのあるトピックの安全な代替は、提供されたアーティファクトの分析であり、ターゲットとのインタラクションではありません。

## 現在のアプリの安全制御

- Safe Labsは静的でローカルな読み取り専用の演習であり、明示的なスコープ契約があります。
- ラボのスコアリングはスコープの承認を必要とし、承認が欠けている場合はペナルティが課されます。
- ラボの証拠は、ライブターゲットの代わりに架空のシステムと文書IP範囲を使用します。
- フラグチャレンジは、提供された静的資産からのみ回答を導き出します。フラグはトレーニングラベルであり、決して認証情報や秘密ではありません。
- フラグの提出は、実際の認証情報、トークン、顧客値、またはプロダクションの秘密を貼り付けることに警告します。
- レポートはスコープを合成とラベル付けし、証拠、影響、修正を求めます。
- 質問パックはインポート前に検証され、ユーザー作成のローカル質問となります。
- フルバックアップと質問パックはローカルJSONファイルであり、アプリはそれらをアップロードしません。
- アプリにはバックエンド、テレメトリ、ネットワークスキャナーはありません。

これらの制御はリスクを軽減しますが、安全でないユーザー作成のコンテンツを受け入れ可能にするものではありません。著者は、インポートされた質問、レポート、ノートをポリシー内に保つ必要があります。

## 禁止コンテンツ

学習者に以下を求めるコンテンツは拒否または書き直してください：

- 実際のホスト、ネットワーク、無線環境、クラウドアカウント、またはサードパーティアプリをスキャンまたは列挙すること。
- ライブの脆弱性を悪用したり、明示的にローカルなおもちゃアプリ以外のシステムに対してペイロードをテストすること。
- マルウェア、ランサムウェア、キーロガー、スパイウェア、ドロッパー、持続性メカニズム、または認証情報を盗むツールを作成、展開、変更、または実行すること。
- フィッシングメッセージを送信したり、認証情報を収集したり、実際のサービスを偽装したり、セッションを収穫すること。
- DoS/DDoSトラフィック、ストレストラフィック、ボットトラフィック、または悪用の自動化を生成すること。
- MFAをバイパスしたり、EDRを打破したり、ログを無効にしたり、活動を隠したり、実際のシステムで権限を昇格させたり、データを外部に持ち出すこと。
- 漏洩したデータセット、個人データ、顧客データ、プロダクションログ、または実際の秘密を使用すること。

## 許可された安全な代替

| リスクのあるトピック | 許可された代替 |
|---|---|
| リコンまたはスキャン | 概念質問、スコープの決定、合成スキャン出力のレビュー、または架空の資産インベントリ分析。 |
| パスワード攻撃 | パスワードポリシーのレビュー、MFAの推論、合成認証ログの分析、またはロックアウト/レート制限設計。 |
| フィッシング | シミュレーションされたヘッダー/ボディ分析、意識トレーニング、検出ルール、または安全な報告ワークフロー。 |
| マルウェア | 静的なおもちゃのインジケーター、ハッシュ/文字列の概念、EDRアラートのレビュー、封じ込め計画、または予防チェックリスト。 |
| ウェブの悪用 | 静的なリクエスト/レスポンスのレビュー、脆弱なコードの推論、またはライブターゲットなしのローカルおもちゃアプリの議論。 |
| 無線攻撃 | 静的PCAP/設定のレビューと防御的強化のみ。 |
| クラウド侵害 | 合成IAM/設定のレビューと最小権限の修正。 |
| DoS/DDoS | キャパシティプランニング、レート制限設計、または合成トラフィックログの分析。 |
| インシデントレスポンス | 架空のインシデントタイムライン、封じ込め、根絶、回復、及び学んだ教訓。 |

## 著作ルール

質問は以下を満たさなければなりません：

- 一度に一つの概念をテストすること。
- 防御的なフレーミングを含むこと：なぜその答えが正しいのか、どの罠を避けるべきか、そしてどの修正またはレビューアクションが続くのか。
- CEHモジュール番号1-20、またはCEH+トラックのモジュール0を使用すること。
- コマンド、ペイロード、ターゲットリスト、認証情報、ターゲットとして使用される実際のサービス名、及びステップバイステップの侵害指示を避けること。

ラボは以下を満たさなければなりません：

- 明示的なスコープ契約から始まること。
- 提供された合成アーティファクトのみを含むこと。
- 学習者が開始する前に禁止された行動を明示すること。
- 発見、証拠、影響、及び修正を求めること。
- モデル回答は分析/修正レベルに保つこと。

レポートは以下を満たさなければなりません：

- 合成スコープを明記すること。
- 提供された証拠のみを引用すること。
- 影響をトレーニングシナリオとして説明し、実際の第三者に対する主張としては扱わないこと。
- 防御的な修正と検証ステップを推奨すること。

## インポートとエクスポートの衛生

進捗バックアップまたは質問パックをインポートする前に：

- JSONが信頼できるソースから来たことを確認すること。
- 質問に実際の認証情報、プライベートデータ、ライブターゲット、または運用攻撃指示が含まれていないことを確認すること。
- インポートされたレポートとノートはローカルトレーニング資料としてのみ扱うこと。

エクスポートする前に：

- 偶発的な秘密、名前、顧客の詳細、または実際のインシデントアーティファクトを削除すること。
- 作成した質問を共有する際は質問パックを優先すること；フル進捗バックアップには個人の学習履歴、レポート、間違い、設定が含まれる可能性があります。

## 安全でないコンテンツの取り扱い

質問、ラボ、レポート、またはインポートされたパックがこのポリシーに違反している場合は、それを公開または共有しないでください。合成証拠を使用し、防御的な推論を求める安全な代替に書き直すか、ローカルアプリから削除してください。

## リリースレビューゲート

リリースを公開したり、デフォルトコンテンツを追加する前に、以下の最小チェックを含む安全レビューを実施してください：

- すべてのシード質問を`node scripts/validate_questions.mjs`で検証すること。
- 読み取り専用コンテンツの安全スキャンを`node scripts/safety_scan.mjs`で実行すること。
- すべてのラボ証拠が合成、ローカル、文書範囲、または架空のデータを使用していることを確認すること。
- UIとREADMEのコピーが認可された防御的使用、合成/ローカルスコープ、及び実際のターゲットがないことを確認すること。
- 公開エクスポートが生の回答、プライベートノート、レポート証拠、認証情報、及びライブターゲットの詳細を除外していることを確認すること。
- 残存リスクと承認を`docs/SECURITY_LEGAL_REVIEW.md`に記録すること。

インポートされたまたはユーザー作成の質問パックは、同じレビューを通過するまでリリースコンテンツではありません。
