# サイバーパンク生産性デザインシステム

## デザイン原則

サイバーパンクは雰囲気。問題管理と復習管理が主役。

```text
Neon where it guides.
Dark where it supports.
Calm where users read.
```

## ワイヤーフレーム

### 質問コンソール

```text
ページヘッダー: モジュール、タグ、難易度、ピン/アーカイブアクション
┌──────────────────────────────┬──────────────────────┐
│ 質問本文                    │ サイドインスペクター  │
│ 選択肢 / 自由形式の回答     │ モジュール、試行、期限 │
│ 提出 / 次のアクション       │ 関連するドリル        │
├──────────────────────────────┴──────────────────────┤
│ 説明: 回答、理由、トラップ、記憶フレーズ               │
└──────────────────────────────────────────────────────┘
```

優先順位: プロンプトと選択肢をまず読みやすく保つ; メタデータは二次的。

### レビューキュー

```text
ページヘッダー: 期限カウント、バックログ、日次目標
┌──────────────────────────────────────────────────────┐
│ 現在のレビューカード                                │
│ 質問 -> 回答 -> 説明                                │
│ 再度 / 難しい / 良い / 簡単な信頼度コントロール      │
├──────────────────────────────┬───────────────────────┤
│ セッション進捗             │ ミスノートショートカット │
└──────────────────────────────┴───────────────────────┘
```

優先順位: 迅速なリコール評価、低い動き、カード間のレイアウトシフトなし。

### 模擬試験

```text
上部バー: タイマー、回答済みカウント、フラグカウント、提出
┌──────────────────────────────┬──────────────────────┐
│ タイムド質問サーフェス     │ 質問ナビゲーター      │
│ 回答コントロール            │ フラグとステータスマップ │
│ 前の / 次の                │                      │
└──────────────────────────────┴──────────────────────┘
```

優先順位: デスクトップファーストのタイムフローで、未回答およびフラグ付きの質問の明確な状態。

### ラボレポート

```text
ページヘッダー: ラボタイトル、スコープステータス、スコア
┌──────────────────────────────┬──────────────────────┐
│ 合成証拠                   │ スコープ契約         │
│ 目的とガイドQ&A            │ 許可された / 禁止された │
├──────────────────────────────┴──────────────────────┤
│ 発見エディタ: 深刻度、影響、修正                       │
│ Markdownコピー/ダウンロード                           │
└──────────────────────────────────────────────────────┘
```

優先順位: スコープの認識と合成証拠は、レポートエクスポートの前に常に表示される必要がある。

## サイバーパンクルール

- Neon glowはアクティブ状態、フォーカス、危険/安全状態だけに使う。
- 問題文、解説、コード、ログにはグリッチアニメーションを使わない。
- サウンドはデフォルトオフ。
- 動きの削減 / 低い光 / 高コントラストを必須にする。
- シティマップは補助的な視覚化。主要導線は生産性UIに置く。
