# NeonSec Academy

[![CI](https://github.com/hjosugi/neonsec-academy/actions/workflows/ci.yml/badge.svg)](https://github.com/hjosugi/neonsec-academy/actions/workflows/ci.yml)

NeonSec Academyは、クライアントサイドのCEH学習および安全な実践トレーニングアプリです。CEHにマッピングされた質問バンク、間隔反復、模擬試験、準備分析、間違いレビュー、合成ラボ、レポート実践をサイバーパンクな学習インターフェースで組み合わせています。

ライブデモ: <https://hjosugi.github.io/neonsec-academy/>（`main`からGitHub Actions経由で公開）

**安全使用ポリシー:** NeonSec Academyは、認可された防御学習のために、ローカルまたは合成素材のみを使用します。このアプリは、ターゲットをスキャンしたり、サードパーティシステムに接触したり、実際の資格情報を使用したり、マルウェアを実行したり、フィッシングメッセージを送信したり、攻撃トラフィックを生成したりしません。コンテンツを作成またはインポートする前に、完全なポリシーをお読みください: [docs/SAFETY_BOUNDARIES.md](docs/SAFETY_BOUNDARIES.md)。

## 製品概要

NeonSec Academyを使用して、練習問題に回答し、弱点をレビューし、試験のタイミングをシミュレートし、実際のシステムに触れることなくセキュリティレポートを実践できます。このアプリは完全にクライアントサイドであり、進捗、設定、カスタム質問、レポート、ラボの進捗は、エクスポートしない限りブラウザストレージに保存されます。

現在の製品範囲、MVPの境界、ペルソナ、成功指標、非目標は[docs/PRODUCT_SCOPE.md](docs/PRODUCT_SCOPE.md)に文書化されています。簡単に言うと、MVPにはCEH質問管理、レビュースケジューリング、模擬試験、分析、安全な合成ラボ、レポート、ローカルインポート/エクスポートが含まれます。ホスティングアカウント、テレメトリ、公式試験ダンプ、実際のターゲットとのインタラクション、エクスプロイトの実行、フィッシングの配信、マルウェア、資格情報の取り扱い、トラフィック生成は明示的に除外されています。

| エリア | 現在のリリース動作 |
|---|---|
| 質問バンク | すべての20のCEHモジュールにわたる371のシード質問とCEH+実践トラック。本文/タグ/モジュールで検索し、モジュール/ドメイン/難易度/タイプでフィルタリングし、質問をピン留めし、カスタム質問をクローン/編集し、ユーザー作成パックをインポート/エクスポートします。 |
| 練習 | すべてのモジュール、特定のCEHモジュール、CEH+トラック、または弱点をターゲットにします。回答はレビュー項目と分析信号を生成します。 |
| レビューキュー | SM-2スタイルの間隔反復スケジュールにより、質問が出され、再度、難しい、良い、または簡単として評価できます。 |
| 模擬試験 | フル試験（125質問/240分）、ハーフレングス（63/120）、クイックシム（25/45）、弱点フォーカス（40/70）で、ドメインの重み付け、フラグ、再開、ドメインごとの結果があります。 |
| 分析 | モジュール/ドメインの習熟度、フラグチャレンジの完了と正確性、最も弱いモジュール、期限のバックログ、準備スコア、バッジ、ストリーク、XP。 |
| 間違いノート | なぜ回答が間違っていたのか、正しい理由、トラップパターン、記憶フレーズ、次のアクション、解決状態を記録します。 |
| 安全なラボ | 持続的な試行とヒントを持つ6つの合成、読み取り専用のフラグチャレンジ: SOCの疑わしいログイン、クラウドIAMレビュー、ウェブアクセス制御、平文資格情報、フィッシングヘッダー、STRIDE脅威モデリング。 |
| 証拠ボールト | チャレンジにリンクされた観察、ログの抜粋、スクリーンショット/ファイルの参照、およびローカル専用の持続性と明示的な機密データ警告を持つノート。 |
| レポート | 範囲、要約、重大度、影響、修正、リンクされたボールトの引用、Markdownコピー、Markdownダウンロードを含む所見レポート。 |
| 最終ゲート | 模擬スコアストリーク、期限のバックログ、弱いモジュールの数、CEHモジュールのカバレッジのための事前予約チェックリストで、Markdownとしてエクスポート可能。 |
| シティマップ | 20のCEHモジュールとCEH+トラックがNeon Tokyo-7地区にマッピングされており、ナビゲーションと進捗スキャンが可能です。 |

## クイックスタート

```bash
# Node 18+が必要です
npm install
npm run dev          # http://localhost:5173
```

プロダクションバンドルをビルドしてプレビューします:

```bash
npm run build        # 型チェックを行い、その後dist/にビルドします
npm run preview
```

シード質問バンクを検証します:

```bash
npm run validate:content
```

## 使用方法

1. アプリを開き、オンボーディングモーダルを完了します。
2. ダッシュボードを使用して次の学習アクションを選択します: 期限の質問をレビューする、練習する、または模擬試験を開始する。
3. 質問バンクを使用して、質問を検索、フィルタリング、ピン留め、クローン、または作成します。
4. レビューキューを毎日使用します; これはあなたの回答とリコールグレードによって駆動されます。
5. 模擬試験を使用してタイムド試験の練習を行い、分析を使用してモジュール/ドメインの弱点を確認します。
6. 安全なラボを使用してトレーニングフラグを提出し、修正を解除し、証拠ボールトに合成観察をキャプチャし、レポートの所見からそれらを引用します。
7. 試験日を予約する前に最終ゲートを使用します; 学習の引き継ぎが必要なときにそのチェックリストをエクスポートします。
8. 設定を使用して、完全な進捗バックアップまたはポータブルなユーザー作成質問パックをエクスポート/インポートします。

## データとプライバシー

NeonSec Academyにはバックエンドもテレメトリもありません。ブラウザストレージには、エクスポートしない限り、ローカル進捗の唯一のコピーが含まれています。完全なバックアップには、プロフィール、設定、質問とフラグの試行、フラグヒントの使用、レビュー状態、間違いノート、アーカイブされた質問ID、カスタム質問、試験結果、証拠ボールトのエントリ、およびレポートが含まれます。質問パックには、ユーザー作成の質問のみが含まれます。

設定は、集計進捗共有のための公開安全なMarkdownエクスポートも提供します。生の回答、提出されたフラグ、カスタム質問テキスト、間違いノート、レポート/ボールト証拠、完全なバックアップデータは除外されます。

実際の資格情報、実際の顧客データ、サードパーティの秘密、プロダクションログ、またはライブターゲットの詳細をカスタム質問、ラボ、レポート、ノート、またはインポートされたJSONに入れないでください。

## デプロイ

アプリは相対的なアセットパスと`HashRouter`を持つ静的サイトであるため、任意の静的ホストが機能します:

```bash
npm run build
# その後、dist/をGitHub Pages、Netlify、Vercel、Cloudflare Pages、または他の静的ホストに公開します
```

## テックスタック

- Vite 8 + React 19 + TypeScript
- 永続化されたブラウザ状態を持つZustand
- React Router `HashRouter`
- カスタムCSSデザインシステムとSVGチャート
- バックエンドなし、テレメトリなし

## プロジェクト構造

```text
src/
  data/              分類法、シード質問、安全なラボ
  data/questions/    group-aからgroup-lのJSONシードパック
  lib/               SRS、試験エンジン、分析、準備、評価、Markdown、質問パック
  store/             永続化されたアプリ状態と派生セレクタ
  components/        UI、チャート、レイアウト、質問ランナー
  routes/            ダッシュボード、質問バンク、練習、レビュー、試験、分析、ラボ、レポート、設定
  styles/            トークン、ベース、サイバー効果、コンポーネント、レイアウト
docs/                リリース、安全、作成、スキーマ、CEHマッピング、システムノート
issues/              段階的実装の問題バックログ
metadata/            ラベル、マイルストーン、プロジェクトボード、問題インデックス
scripts/             コンテンツ検証とGitHubインポートヘルパー
```
