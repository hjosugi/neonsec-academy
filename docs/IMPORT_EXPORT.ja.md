# インポートとエクスポート

NeonSec Academyは、ブラウザ内に状態をローカルに保存します。インポート/エクスポートは、バックアップ、デバイストランスファー、質問共有、Markdownの引き渡しのためのものです。このアプリはファイルをアップロードしたり、バックエンドと同期したりすることはありません。

すべてのインポートされたコンテンツは、[SAFETY_BOUNDARIES.md](SAFETY_BOUNDARIES.md)に従う必要があります。実際の認証情報、プロダクションログ、顧客データ、個人データ、ライブターゲットの詳細、エクスプロイトチェーン、またはその他の危険な素材をインポートまたは共有しないでください。

## エクスポートタイプ

| エクスポート | ファイル名 | フォーマット | 使用時 |
|---|---|---|---|
| 完全な進捗バックアップ | `neonsec-backup.json` | アプリ状態JSON, `version: 1` | 自分の学習状態をバックアップまたは移動したいとき。 |
| 公共安全な進捗サマリー | `neonsec-public-progress.md` | Markdown | プライベートノート、生の回答、カスタム質問テキスト、または報告/ボールトの証拠を共有せずに集計進捗を共有したいとき。 |
| 質問パック | `neonsec-question-pack.json` | `neonsec-question-pack`, `version: 1` | 個人の進捗を共有せずにユーザー作成の質問を共有したいとき。 |
| 質問JSONL | `neonsec-question-pack.jsonl` | 各行に1つの`RawQuestion` JSONオブジェクト | 行指向の編集、差分取得、またはバッチインポート/エクスポートを行いたいとき。 |
| 質問CSV | カスタム`.csv` | ヘッダー行とMCQ行のみ | 基本的な選択肢問題のスプレッドシートインポートを行いたいとき。 |
| レポートMarkdown | カスタムレポートタイトルとして`.md` | Markdown | レポートからの所見レポートの引き渡しを行いたいとき。 |
| 最終ゲートMarkdown | `ceh-final-gate-checklist.md` | Markdown | 予約前のチェックリストのスナップショットを取得したいとき。 |

## 完全な進捗バックアップ

アプリ内で：

1. 設定を開く。
2. **完全バックアップJSONをエクスポート**を使用する。
3. `neonsec-backup.json`をプライベートな場所に保存する。

バックアップには以下が含まれます：

- `version`
- `exportedAt`
- `profile`
- `settings`
- `attempts`
- `reviews`
- `mistakes`
- `bookmarks`
- `archivedIds`
- `userQuestions`
- `examResults`
- `flagAttempts`
- `flagHintUses`
- `evidenceItems`
- `reports`

完全バックアップは、自分のデバイスの移行や復元に使用します。公開しないでください。個人の学習履歴、提出されたフラグ、ヒントの使用、カスタムミスノート、証拠ボールトのエントリ、レポート、設定、作成した質問が含まれる可能性があります。

## 完全な進捗のインポート

アプリ内で：

1. 設定を開く。
2. **完全バックアップJSONをインポート**を使用する。
3. JSONバックアップを選択する。
4. アプリが成功したインポートメッセージを表示することを確認する。

インポートはJSONオブジェクトを受け入れ、欠落しているプロファイル/設定フィールドを現在のデフォルトとマージします。オプションの配列は、配列として存在する場合にのみインポートされます。それ以外の場合は、現在のローカル値が保持されます。フラグの試行とヒントの使用は、現在の静的チャレンジに対しても正規化されます：不明な行は破棄され、ヒントインデックスは重複排除され、正確性が再計算されます。これにより、古いまたは部分的なバックアップを安全に読み込むことができますが、インポートされたデータは読み込み後にレビューする必要があります。

インポート前に：

- 手動でロールバックする必要があるかもしれない場合は、まず現在の進捗をエクスポートしてください。
- 読み込む前に信頼できないJSONを検査してください。
- ファイルに秘密、個人データ、プロダクションアーティファクト、または危険なコンテンツが含まれていないことを確認してください。

## 公共安全な進捗サマリー

学習履歴の詳細を公開せずに進捗を共有したいときは、公共安全なエクスポートを使用してください。

アプリ内で：

1. 設定を開く。
2. **プライバシーエクスポート**のプレビューを確認する。
3. プライバシーチェックリストを完了する。
4. **公共安全なMarkdownをエクスポート**を使用する。

Markdownには以下が含まれます：

- 集計された試行回数、正解数、正確性。
- モード別にグループ化された試行。
- レビューアイテムのカウントと期限/スケジュール/保留のサマリー。
- 模擬試験のサマリー。
- ラボの進捗サマリー。
- 深刻度カウントを含むレポートタイトルのサマリー。

Markdownには以下が含まれません：

- 生の選択された回答と質問ID。
- 提出されたフラグ値とヒントごとの使用履歴。
- カスタム質問テキストと回答。
- ミスノートのノート。
- レポートの範囲、サマリー、影響、修正、証拠。
- 証拠ボールトのタイトル、ノート、ソース、ファイル/スクリーンショットの参照、およびタイムスタンプ。
- 完全バックアップJSON。

一般的な敏感なレポートタイトルパターンは、エクスポート前にマスクされます：メールアドレス、IPv4アドレス、単純な`password`、`passwd`、`pwd`、`token`、`secret`、または`api_key`の割り当て。

## 質問パック

質問パックは、ユーザー作成の質問の共有可能なフォーマットです。試行、レビュー、ミスノート、試験結果、レポート、設定、またはプロファイルデータは含まれません。設定は、3つの質問インポートフォーマットをサポートしています：

- JSONパックオブジェクト：構造化されたデッキメタデータと`questions[]`。
- JSONL：空でない各行に1つの有効な質問JSONオブジェクト。
- CSV：基本的な列を持つMCQ専用のスプレッドシートインポート。

パックの形状：

```json
{
  "format": "neonsec-question-pack",
  "version": 1,
  "title": "NeonSecカスタム質問 (2)",
  "exportedAt": 1783526400000,
  "questions": []
}
```

`questions`内の各アイテムは、[QUESTION_SCHEMA.md](QUESTION_SCHEMA.md)を満たす必要があります。インポートされた質問は、`source: "user"`および`status: "active"`に正規化されます。

JSONLの形状：

```jsonl
{"id":"Q-USER-1","type":"mcq","module":1,"track":null,"difficulty":"easy","tags":["scope"],"body":"どのターゲットが範囲内ですか？","choices":["承認されたホスト","任意のホスト"],"answer":"承認されたホスト","explanation":{"answer":"承認されたホスト","why":"書面による範囲が権限を制御します。","trap":"公共の到達可能性は許可ではありません。","memory_phrase":"まず範囲を確認してください。"}}
```

CSVインポートは`mcq`に制限されています。必須列：

```csv
id,title,module,track,difficulty,tags,body,choice_a,choice_b,choice_c,choice_d,answer,explanation_answer,explanation_why,explanation_trap,memory_phrase
```

`tags`は`;`または`|`区切りを使用します。`answer`は正確な選択肢のテキストまたは`A`から`F`の文字である場合があります。検証に失敗したCSV行は、ソース行番号を報告します。

## 質問パックのエクスポート

アプリ内で：

1. 設定を開く。
2. **質問パック**で、作成した質問を選択するか、**すべて選択**を使用する。
3. JSON用に**選択したパックをエクスポート** / **すべての作成した質問をエクスポート**を使用するか、行指向のエクスポート用にJSONLボタンを使用する。
4. 結果の`.json`または`.jsonl`を、安全な使用ポリシーに従っている場合のみ共有する。

ユーザー作成の質問のみがパックとしてエクスポート可能です。シード質問はアプリのビルドの一部として残り、このワークフローを通じてエクスポートされることはありません。

## 質問パックのインポート

アプリ内で：

1. 設定を開く。
2. **JSON / JSONL / CSVをインポート**を使用する。
3. JSONパック、JSONLファイル、またはMCQ CSVを選択する。
4. プレビューを確認する：合計質問数、モジュール、難易度のカウント、IDの衝突。
5. コンテンツが安全で予想通りである場合のみ、**パックをインポート**を選択する。

インポータは以下を拒否します：

- 無効なJSON。
- サポートされていない`format`または`version`。
- 欠落している`questions`。
- 無効な質問フィールド。
- パック内の重複ID。
- 選択肢と一致しない回答。
- 無効なJSONL/CSV行、行番号と共に報告されます。

パックの質問IDがすでにローカルに存在する場合、インポータは保存前に`-import-N`サフィックスで再マッピングします。プレビューは、いくつの衝突が名前変更されるかを警告します。

## Markdownエクスポート

レポートは、レポート画面からMarkdownをエクスポートします。レポートは空であるか、手動で作成されるか、安全なラボからシードされることがあります。所見は、チャレンジに一致する証拠ボールトアイテムにリンクできます。エクスポートされたレポートは、証拠の種類、タイトル、ソース、キャプチャ時間、オプションのファイル/スクリーンショットの参照、およびノートを含む引用にそれらのリンクを解決します。エクスポートされたレポートには、タイトル、合成された範囲、サマリー、所見、影響、修正、手動証拠ノート、および生成者のフッターも含まれます。インポートとブラウザの水和は、無効なボールト行を破棄し、引用IDを重複排除し、レポートビルダーに到達する前に欠落またはクロスチャレンジリンクを削除します。

模擬試験結果は、試験レポート画面からMarkdownをエクスポートします。スナップショットには、スコア、ターゲット、安全マージン、費やした時間、フラグ付きの正確性、モジュールの内訳、および次の7日間の修正計画が含まれます。

最終ゲートは、最終ゲート画面からMarkdownをエクスポートします。チェックリストには、合格/不合格のステータス、設定された基準、最近の模擬結果、弱いモジュールの警告、および次のアクションが含まれます。

Markdownエクスポートはスナップショットです。Markdownをアプリに再インポートすることはサポートされていません。

## 安全な共有チェックリスト

- [ ] ファイルに秘密、実際の認証情報、トークン、クッキー、キー、またはプライベートURLが含まれていない。
- [ ] ファイルに顧客データ、個人データ、プロダクションログ、または実際のインシデントアーティファクトが含まれていない。
- [ ] 質問は、安全なCEH認識、推論、検出、修正、または範囲のプロンプトのみを使用している。
- [ ] ラボ/レポートは、合成証拠と防御的推奨を説明している。
- [ ] 完全な進捗バックアップは、意図的に信頼できる誰かと共有しない限りプライベートに保たれる。
- [ ] 質問パックは、作成した質問を共有するために好まれる。
